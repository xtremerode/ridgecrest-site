## Known Issues Log

All issues discovered during the April 9, 2026 build session. Review before running any script.

| # | Issue | Root Cause | Resolution |
|---|---|---|---|
| 1 | 94575 not targetable | Google does not have 94575 as a US postal code geo target | Removed; only 17 zip codes used |
| 2 | Ampersand (&) in ad copy triggers PROHIBITED policy | Google Ads policy flags & as prohibited character in ad text | Use "and" instead of & in all ad copy |
| 3 | Phone number in descriptions triggers PROHIBITED policy | Google Ads policy prohibits phone numbers in description text | Removed phone number from all descriptions |
| 4 | containsEuPoliticalAdvertising: false causes enum error | false is not a valid enum value for this field | Use string "DOES_NOT_CONTAIN_EU_POLITICAL_ADVERTISING" |
| 5 | Newly created campaign cannot be queried in same script run | Google Ads indexing lag — new objects are not immediately queryable | Split into Script 1 (create) and Script 2 (configure); run separately |
| 6 | Temporary resource names (-1, -2, -3) do not persist after mutateAll | Resource names are session-scoped and not reusable across scripts | Never reference temp resource names in subsequent scripts; always query for real IDs |
| 7 | Google Ads Scripts run in Preview mode by default | Default mode — changes are simulated, not applied | Must click Run (not just Preview) to make real changes |
| 8 | Non-ASCII characters cause SyntaxError | Google Ads Scripts JavaScript engine does not accept non-ASCII | Use only standard ASCII in all script files; no em dashes, curly quotes, box-drawing chars |
| 9 | Mixed quote types in strings cause SyntaxError | JavaScript parser rejects inconsistent quote usage | Use only double quotes throughout all scripts |
